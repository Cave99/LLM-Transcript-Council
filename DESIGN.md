# Design

## Theme

Local evaluation workbench used by technical teams during focused prompt/model iteration on a laptop or external monitor. Use a light, quiet interface because the tool is used while reading long transcripts and JSON outputs.

## Color

Use restrained OKLCH-tinted neutrals with one confident green-blue accent for primary actions and success states. Use amber for running/warning states and red-orange for failures. Avoid pure black and pure white.

## Typography

Use the native system sans stack throughout. Keep product UI labels compact and readable. Reserve larger headings only for page titles.

## Layout

Use a stable app shell, compact page headers, clear sections, and dense but breathable tables. Avoid decorative cards around every page section. Use panels only for forms, data tables, and drill-down blocks.

## Components

Buttons, inputs, tables, status pills, progress meters, and list rows should share one visual vocabulary. Interactive states need hover and focus styles. Empty states should guide the next action.

## Motion

Use short 150-200ms transitions for hover/focus/state changes only. Respect reduced-motion preferences.
