# Product

## Register

product

## Users

DS and AI engineers evaluating prompt, model, and temperature configurations for call-transcript LLM systems. They are technical users working locally, often comparing subjective outputs where a labelled dataset is unavailable or too expensive to maintain.

## Product Purpose

LLM-Transcript-Council helps teams compare subjective LLM outputs through blind pairwise judging, judge councils, swapped A/B validation, and ELO rankings. Success means a user can quickly create a reproducible evaluation run, monitor progress, and explain why one configuration tends to outperform another.

## Brand Personality

Calm, rigorous, inspectable. The interface should feel like a serious evaluation workbench: precise, low-friction, and trustworthy without looking sterile.

## Anti-references

Avoid default Streamlit dashboards, decorative SaaS landing pages, oversized hero marketing sections, purple-gradient AI tooling, playful gamified ranking screens, and dense enterprise admin UI that hides the evaluation story.

## Design Principles

- Make subjective comparison feel auditable.
- Put configuration and evidence close together.
- Show progress without drama.
- Prefer calm density over decoration.
- Keep the first run obvious for technical users.

## Accessibility & Inclusion

Target WCAG AA contrast, visible keyboard focus, readable table and form text, reduced-motion support, and layouts that remain usable on laptop-sized screens.
